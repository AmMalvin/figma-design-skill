# Imagery & Illustration

## Purpose

This module teaches the AI how to build a consistent visual language using photography, illustrations, graphics, avatars, and supporting visual assets.

Every image should communicate purpose.

Visual assets should strengthen usability before aesthetics.

---

# Definition

Imagery and illustration define the visual storytelling system of a product.

They communicate information, reinforce branding, support recognition, and improve emotional engagement.

Every visual asset should belong to one unified system.

---

# Objectives

After completing this module, the AI should be able to:

- Build consistent imagery systems
- Create illustration guidelines
- Improve visual storytelling
- Strengthen branding
- Improve accessibility
- Standardize image usage
- Create reusable asset libraries
- Improve developer implementation

---

# Visual Mindset

Never ask:

"Does this image look beautiful?"

Always ask:

- Does it communicate clearly?
- Does it support the user task?
- Is it consistent with the design system?
- Is it accessible?
- Can it be reused?

---

# Core Principles

Always:

- Prioritize communication.
- Use authentic visuals.
- Maintain consistency.
- Support accessibility.
- Optimize performance.
- Build reusable assets.

Never:

- Use decorative images without purpose.
- Mix illustration styles.
- Distort images.
- Ignore accessibility.
- Use inconsistent visual quality.

---

# Why Visual Assets Exist

Visual assets should:

- Explain
- Guide
- Educate
- Reinforce branding
- Build trust
- Improve recognition
- Support onboarding
- Improve emotional connection

Visuals should never distract from primary tasks.

---

# Visual Asset Categories

Every product should define:

- Photography
- Illustration
- Graphics
- Avatars
- Empty States
- Onboarding Artwork
- Marketing Assets
- Product Images
- Background Graphics
- Decorative Assets

Each category should have documented rules.

---

# Visual Consistency

Every visual asset should maintain consistency in:

- Style
- Perspective
- Color treatment
- Lighting
- Composition
- Quality
- Resolution
- Cropping

Consistency improves product recognition.

---

# Brand Expression

Imagery should reinforce:

- Brand personality
- Product values
- User expectations
- Emotional tone

Visual identity should remain recognizable across every screen.

---

# AI Decision Rules

Before approving any visual asset, answer:

- Does it communicate clearly?
- Does it belong to the visual system?
- Is accessibility supported?
- Is quality sufficient?
- Can it be reused?
- Does it strengthen the brand?

If any answer is negative, redesign the asset.

---

# Validation Checklist

□ Visual language defined

□ Asset categories documented

□ Brand consistency reviewed

□ Accessibility reviewed

□ Reusability confirmed

□ Documentation updated

# Photography

## Definition

Photography is the use of real-world images to communicate information, build trust, and reinforce the product's identity.

Photography should always support user understanding.

It should never exist purely for decoration.

---

# Photography Principles

Photography should be:

- Authentic
- Relevant
- High quality
- Inclusive
- Consistent
- Purposeful

Every image should support a user goal.

---

# Photography Style

Every product should define one photography style.

Maintain consistency in:

- Perspective
- Lighting
- Contrast
- Color treatment
- Subject distance
- Background
- Composition

Never mix conflicting photography styles.

---

# Composition

Composition determines how visual elements are arranged.

Good composition should:

- Guide attention
- Highlight important subjects
- Remove distractions
- Improve storytelling

Composition should support the interface, not compete with it.

---

# Cropping

Crop images consistently.

Avoid:

- Cutting important subjects
- Inconsistent framing
- Distorted aspect ratios

Cropping should preserve context.

---

# Lighting

Lighting should remain consistent across the product.

Preferred characteristics:

- Natural
- Soft
- Even
- Clear

Avoid dramatic lighting unless required by the brand.

---

# Backgrounds

Backgrounds should support the subject.

Preferred backgrounds:

- Simple
- Clean
- Low distraction

Backgrounds should never reduce readability.

---

# Human Representation

People should be represented naturally.

Images should reflect:

- Real situations
- Genuine expressions
- Appropriate environments

Avoid exaggerated or unrealistic poses.

---

# Diversity and Inclusion

Photography should represent diverse users.

Consider diversity across:

- Age
- Gender
- Ethnicity
- Ability
- Culture

Representation should feel authentic.

Avoid stereotypes and token representation.

---

# Illustration

## Definition

Illustrations communicate ideas that photography cannot easily express.

Illustrations should simplify concepts rather than increase complexity.

---

# Illustration Principles

Illustrations should:

- Explain
- Guide
- Reinforce branding
- Reduce complexity

Illustrations should never replace essential interface content.

---

# Illustration Styles

Every design system should define one primary illustration style.

Examples include:

- Flat
- Isometric
- Three-dimensional
- Hand-drawn
- Geometric
- Minimal

Choose one primary style.

Avoid mixing styles throughout the product.

---

# Flat Illustrations

Characteristics:

- Simple shapes
- Minimal depth
- Clear communication

Recommended for:

- SaaS
- Dashboards
- Enterprise software
- Productivity tools

---

# Isometric Illustrations

Characteristics:

- Three-dimensional perspective
- Structured layouts
- Technical appearance

Recommended for:

- Developer products
- Infrastructure platforms
- Data visualization

Use consistently.

---

# Three-Dimensional Illustrations

Characteristics:

- Depth
- Shadows
- Volume

Recommended for:

- Marketing
- Product launches
- Consumer experiences

Use sparingly inside productivity interfaces.

---

# Hand-Drawn Illustrations

Characteristics:

- Organic
- Friendly
- Informal

Recommended for:

- Education
- Children
- Creative products

Avoid mixing with rigid geometric illustrations.

---

# Visual Consistency

Every illustration should maintain:

- Perspective
- Stroke style
- Corner treatment
- Color palette
- Shadow treatment
- Detail level

Illustrations should feel like one family.

---

# AI Visual Style Engine

Before approving photography or illustrations, answer:

- Does the visual communicate clearly?
- Is the style consistent?
- Does it support the product?
- Is representation inclusive?
- Does it strengthen the brand?
- Can this style scale across every screen?

If any answer is negative, redesign the visual system.

# Empty States

## Definition

Empty states communicate the absence of content while guiding users toward the next action.

Every empty state should include:

- A clear illustration or image
- A descriptive title
- Supporting text
- A primary action when appropriate

Empty states should reduce uncertainty.

---

# Empty State Principles

Empty states should:

- Explain why nothing appears
- Suggest the next step
- Reduce frustration
- Maintain a positive tone

Avoid decorative artwork without guidance.

---

# Onboarding Illustrations

Onboarding illustrations introduce users to the product.

They should:

- Reinforce key concepts
- Reduce cognitive load
- Match the product's visual language
- Support learning

Illustrations should never replace clear explanations.

---

# Product Graphics

Product graphics explain features and workflows.

Examples include:

- Feature diagrams
- Workflow graphics
- Product previews
- Process illustrations

Every graphic should communicate one primary idea.

---

# Marketing Visuals

Marketing visuals should strengthen brand recognition.

Maintain consistency in:

- Composition
- Typography
- Color usage
- Illustration style
- Photography style

Marketing assets should remain visually connected to the product.

---

# Hero Images

Hero images introduce major pages.

Hero images should:

- Establish context
- Highlight value
- Support messaging
- Avoid unnecessary complexity

The subject should remain immediately recognizable.

---

# Background Graphics

Background graphics provide visual support.

Background graphics should:

- Remain subtle
- Never reduce readability
- Avoid competing with interface elements

Decorative backgrounds should have a supporting role.

---

# Avatars

Avatars represent people, organizations, or entities.

Avatar systems should define:

- Shape
- Sizes
- Placeholder behavior
- Fallback initials
- Default imagery

Avatar styles should remain consistent throughout the product.

---

# Thumbnails

Thumbnails provide quick visual previews.

Maintain consistency in:

- Aspect ratio
- Cropping
- Border radius
- Alignment
- Resolution

Every thumbnail should remain recognizable at small sizes.

---

# Image Placeholders

Placeholders communicate that content is loading or unavailable.

Placeholders should:

- Match final image dimensions
- Preserve layout stability
- Minimize perceived loading time

Avoid layout shifts when images load.

---

# Aspect Ratios

Images should follow predefined aspect ratios.

Common ratios include:

- 1:1
- 3:2
- 4:3
- 16:9
- 9:16

Select one ratio based on content requirements.

Do not distort images.

---

# Image Optimization

Optimize every image before publication.

Optimize:

- Resolution
- File size
- Compression
- Export format

Images should balance quality with performance.

---

# Responsive Images

Images should adapt across devices.

Responsive behavior should preserve:

- Subject visibility
- Aspect ratio
- Composition
- Readability

Cropping should remain intentional across breakpoints.

---

# Asset Variants

Visual assets may require multiple variants.

Examples:

- Light
- Dark
- High Contrast
- Mobile
- Desktop

Variants should remain visually consistent.

---

# Asset Naming

Use descriptive names.

Examples:

hero-dashboard

empty-search

illustration-onboarding-01

avatar-default

thumbnail-course

Avoid names such as:

image1

graphic-final

new-image

Every asset name should describe its purpose.

---

# AI Asset Composition Engine

Before approving a visual asset, answer:

- Does the asset support the user task?
- Is composition balanced?
- Is the aspect ratio correct?
- Is optimization complete?
- Is accessibility maintained?
- Does the asset belong to the visual system?
- Can it be reused across products?

If any answer is negative, redesign the asset.
