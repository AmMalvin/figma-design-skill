# Imagery & Illustration

## Purpose

This module teaches the AI how to build a consistent visual language using photography, illustrations, graphics, avatars, and supporting visual assets.

Every image should communicate purpose.

Visual assets should strengthen usability before aesthetics.

---

# Definition

Imagery and illustration define the visual storytelling system of a product.

They communicate information, reinforce branding, support recognition, and improve emotional engagement.

Every visual asset should belong to one unified system.

---

# Objectives

After completing this module, the AI should be able to:

- Build consistent imagery systems
- Create illustration guidelines
- Improve visual storytelling
- Strengthen branding
- Improve accessibility
- Standardize image usage
- Create reusable asset libraries
- Improve developer implementation

---

# Visual Mindset

Never ask:

"Does this image look beautiful?"

Always ask:

- Does it communicate clearly?
- Does it support the user task?
- Is it consistent with the design system?
- Is it accessible?
- Can it be reused?

---

# Core Principles

Always:

- Prioritize communication.
- Use authentic visuals.
- Maintain consistency.
- Support accessibility.
- Optimize performance.
- Build reusable assets.

Never:

- Use decorative images without purpose.
- Mix illustration styles.
- Distort images.
- Ignore accessibility.
- Use inconsistent visual quality.

---

# Why Visual Assets Exist

Visual assets should:

- Explain
- Guide
- Educate
- Reinforce branding
- Build trust
- Improve recognition
- Support onboarding
- Improve emotional connection

Visuals should never distract from primary tasks.

---

# Visual Asset Categories

Every product should define:

- Photography
- Illustration
- Graphics
- Avatars
- Empty States
- Onboarding Artwork
- Marketing Assets
- Product Images
- Background Graphics
- Decorative Assets

Each category should have documented rules.

---

# Visual Consistency

Every visual asset should maintain consistency in:

- Style
- Perspective
- Color treatment
- Lighting
- Composition
- Quality
- Resolution
- Cropping

Consistency improves product recognition.

---

# Brand Expression

Imagery should reinforce:

- Brand personality
- Product values
- User expectations
- Emotional tone

Visual identity should remain recognizable across every screen.

---

# AI Decision Rules

Before approving any visual asset, answer:

- Does it communicate clearly?
- Does it belong to the visual system?
- Is accessibility supported?
- Is quality sufficient?
- Can it be reused?
- Does it strengthen the brand?

If any answer is negative, redesign the asset.

---

# Validation Checklist

□ Visual language defined

□ Asset categories documented

□ Brand consistency reviewed

□ Accessibility reviewed

□ Reusability confirmed

□ Documentation updated

# Photography

## Definition

Photography is the use of real-world images to communicate information, build trust, and reinforce the product's identity.

Photography should always support user understanding.

It should never exist purely for decoration.

---

# Photography Principles

Photography should be:

- Authentic
- Relevant
- High quality
- Inclusive
- Consistent
- Purposeful

Every image should support a user goal.

---

# Photography Style

Every product should define one photography style.

Maintain consistency in:

- Perspective
- Lighting
- Contrast
- Color treatment
- Subject distance
- Background
- Composition

Never mix conflicting photography styles.

---

# Composition

Composition determines how visual elements are arranged.

Good composition should:

- Guide attention
- Highlight important subjects
- Remove distractions
- Improve storytelling

Composition should support the interface, not compete with it.

---

# Cropping

Crop images consistently.

Avoid:

- Cutting important subjects
- Inconsistent framing
- Distorted aspect ratios

Cropping should preserve context.

---

# Lighting

Lighting should remain consistent across the product.

Preferred characteristics:

- Natural
- Soft
- Even
- Clear

Avoid dramatic lighting unless required by the brand.

---

# Backgrounds

Backgrounds should support the subject.

Preferred backgrounds:

- Simple
- Clean
- Low distraction

Backgrounds should never reduce readability.

---

# Human Representation

People should be represented naturally.

Images should reflect:

- Real situations
- Genuine expressions
- Appropriate environments

Avoid exaggerated or unrealistic poses.

---

# Diversity and Inclusion

Photography should represent diverse users.

Consider diversity across:

- Age
- Gender
- Ethnicity
- Ability
- Culture

Representation should feel authentic.

Avoid stereotypes and token representation.

---

# Illustration

## Definition

Illustrations communicate ideas that photography cannot easily express.

Illustrations should simplify concepts rather than increase complexity.

---

# Illustration Principles

Illustrations should:

- Explain
- Guide
- Reinforce branding
- Reduce complexity

Illustrations should never replace essential interface content.

---

# Illustration Styles

Every design system should define one primary illustration style.

Examples include:

- Flat
- Isometric
- Three-dimensional
- Hand-drawn
- Geometric
- Minimal

Choose one primary style.

Avoid mixing styles throughout the product.

---

# Flat Illustrations

Characteristics:

- Simple shapes
- Minimal depth
- Clear communication

Recommended for:

- SaaS
- Dashboards
- Enterprise software
- Productivity tools

---

# Isometric Illustrations

Characteristics:

- Three-dimensional perspective
- Structured layouts
- Technical appearance

Recommended for:

- Developer products
- Infrastructure platforms
- Data visualization

Use consistently.

---

# Three-Dimensional Illustrations

Characteristics:

- Depth
- Shadows
- Volume

Recommended for:

- Marketing
- Product launches
- Consumer experiences

Use sparingly inside productivity interfaces.

---

# Hand-Drawn Illustrations

Characteristics:

- Organic
- Friendly
- Informal

Recommended for:

- Education
- Children
- Creative products

Avoid mixing with rigid geometric illustrations.

---

# Visual Consistency

Every illustration should maintain:

- Perspective
- Stroke style
- Corner treatment
- Color palette
- Shadow treatment
- Detail level

Illustrations should feel like one family.

---

# AI Visual Style Engine

Before approving photography or illustrations, answer:

- Does the visual communicate clearly?
- Is the style consistent?
- Does it support the product?
- Is representation inclusive?
- Does it strengthen the brand?
- Can this style scale across every screen?

If any answer is negative, redesign the visual system.
