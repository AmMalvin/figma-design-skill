# Mental Models

## Purpose

This module teaches the AI how users understand digital products through prior knowledge, experience, and expectations.

Users rarely learn every interface from zero.

Instead, they transfer knowledge from products they already know.

The AI must design interfaces that align with existing mental models whenever possible.

---

# Definition

A mental model is the internal representation users build about how a system works.

Mental models are formed through repeated experiences across websites, mobile apps, operating systems, and physical objects.

Users predict behavior based on these models.

Good interfaces match those predictions.

Poor interfaces force unnecessary learning.

---

# Objectives

After completing this module, the AI should be able to:

- Predict user expectations
- Reuse familiar interaction patterns
- Reduce learning time
- Improve discoverability
- Improve learnability
- Reduce confusion
- Build confidence
- Increase consistency

---

# Mental Model Mindset

Never ask:

"Can we invent a new interaction?"

Always ask:

- What do users already expect?
- Which existing pattern solves this problem?
- Will users immediately understand this?
- Does this reduce learning effort?
- Does this improve confidence?

Innovation should solve real problems rather than replace familiar patterns without evidence.

---

# Core Principles

Always:

- Respect user expectations.
- Reuse familiar conventions.
- Maintain consistency.
- Support recognition.
- Explain unavoidable changes.
- Minimize relearning.

Never:

- Reinvent common controls.
- Break platform conventions.
- Hide important functionality.
- Introduce novelty without measurable benefit.

---

# Human Expectations

Users expect:

- Predictable navigation
- Consistent buttons
- Familiar icons
- Clear labels
- Immediate feedback
- Standard gestures
- Logical workflows

Unexpected behavior should require strong evidence before implementation.

---

# Sources of Mental Models

Mental models come from:

- Operating systems
- Web browsers
- Mobile applications
- Physical products
- Previous workplace tools
- Industry standards
- Cultural conventions

Design decisions should acknowledge these influences.

---

# Types of Mental Models

## Existing Models

Knowledge users already possess.

Example:

Shopping cart behavior across e-commerce platforms.

---

## Emerging Models

Patterns becoming common across modern products.

Example:

Command palettes and universal search.

---

## Domain-Specific Models

Patterns unique to a particular industry.

Examples:

Healthcare

Banking

Project Management

Engineering

Professional software may require specialized mental models while maintaining familiar interaction patterns.

---

# AI Decision Rules

Before designing any interaction, answer:

- What will users expect?
- Which mental model applies?
- Is there an established pattern?
- Will users recognize the interaction immediately?
- Does this reduce learning effort?
- Is deviation from convention justified?

If the answer is unclear, prefer established patterns.

# Jakob's Law

## Definition

Users spend most of their time using other digital products.

They expect your product to work similarly to products they already know.

The AI should leverage existing mental models whenever practical.

Innovation should improve user outcomes, not increase learning effort.

---

# Applying Jakob's Law

Always:

- Follow platform conventions.
- Reuse familiar interaction patterns.
- Maintain predictable navigation.
- Keep common actions where users expect them.
- Preserve established terminology.

Avoid changing familiar interactions unless there is measurable evidence that the new approach is superior.

---

# Recognition vs Recall

Recognition requires users to identify visible information.

Recall requires users to retrieve information from memory.

Recognition should always be preferred.

Examples:

Prefer:

- Visible navigation
- Search suggestions
- Auto-complete
- Saved history
- Labeled icons

Avoid:

- Hidden commands
- Memorized shortcuts
- Invisible gestures
- Complex navigation paths

Recognition reduces learning effort.

---

# Transfer Learning

## Definition

Transfer learning occurs when knowledge gained from one product helps users understand another.

Examples:

- Shopping carts
- Search icons
- Profile avatars
- Settings pages
- Bottom navigation
- Breadcrumb navigation

Support transfer learning by maintaining familiar patterns.

---

# Platform Conventions

Users expect different conventions on different platforms.

Respect conventions for:

- Web
- iOS
- Android
- Desktop
- Tablet
- Wearables

Never force one platform to imitate another without strong evidence.

---

# Conceptual Models

A conceptual model explains how a system works.

Users build conceptual models from:

- Navigation
- Labels
- Visual hierarchy
- Feedback
- States
- Documentation

Strong conceptual models allow users to predict outcomes before taking action.

---

# Discoverability

Features should be easy to find.

Improve discoverability through:

- Clear navigation
- Visible controls
- Logical grouping
- Progressive disclosure
- Familiar labels
- Strong hierarchy

Hidden functionality should remain the exception.

---

# Learnability

Interfaces should become easier over time.

Improve learnability through:

- Consistency
- Familiar workflows
- Repetition
- Immediate feedback
- Predictable behavior

Every interaction should reinforce future understanding.

---

# Predictability

Users should accurately predict the result of every action.

Predictability requires:

- Consistent layouts
- Consistent interactions
- Stable navigation
- Clear labels
- Reliable feedback

Unexpected behavior reduces trust.

---

# AI Decision Rules

Before approving any interaction, answer:

- Will users immediately recognize this?
- Does it match established conventions?
- Does it reduce learning effort?
- Does it strengthen the user's mental model?
- Can users predict the outcome?

If not, redesign the interaction.
