# Mental Models

## Purpose

This module teaches the AI how users understand digital products through prior knowledge, experience, and expectations.

Users rarely learn every interface from zero.

Instead, they transfer knowledge from products they already know.

The AI must design interfaces that align with existing mental models whenever possible.

---

# Definition

A mental model is the internal representation users build about how a system works.

Mental models are formed through repeated experiences across websites, mobile apps, operating systems, and physical objects.

Users predict behavior based on these models.

Good interfaces match those predictions.

Poor interfaces force unnecessary learning.

---

# Objectives

After completing this module, the AI should be able to:

- Predict user expectations
- Reuse familiar interaction patterns
- Reduce learning time
- Improve discoverability
- Improve learnability
- Reduce confusion
- Build confidence
- Increase consistency

---

# Mental Model Mindset

Never ask:

"Can we invent a new interaction?"

Always ask:

- What do users already expect?
- Which existing pattern solves this problem?
- Will users immediately understand this?
- Does this reduce learning effort?
- Does this improve confidence?

Innovation should solve real problems rather than replace familiar patterns without evidence.

---

# Core Principles

Always:

- Respect user expectations.
- Reuse familiar conventions.
- Maintain consistency.
- Support recognition.
- Explain unavoidable changes.
- Minimize relearning.

Never:

- Reinvent common controls.
- Break platform conventions.
- Hide important functionality.
- Introduce novelty without measurable benefit.

---

# Human Expectations

Users expect:

- Predictable navigation
- Consistent buttons
- Familiar icons
- Clear labels
- Immediate feedback
- Standard gestures
- Logical workflows

Unexpected behavior should require strong evidence before implementation.

---

# Sources of Mental Models

Mental models come from:

- Operating systems
- Web browsers
- Mobile applications
- Physical products
- Previous workplace tools
- Industry standards
- Cultural conventions

Design decisions should acknowledge these influences.

---

# Types of Mental Models

## Existing Models

Knowledge users already possess.

Example:

Shopping cart behavior across e-commerce platforms.

---

## Emerging Models

Patterns becoming common across modern products.

Example:

Command palettes and universal search.

---

## Domain-Specific Models

Patterns unique to a particular industry.

Examples:

Healthcare

Banking

Project Management

Engineering

Professional software may require specialized mental models while maintaining familiar interaction patterns.

---

# AI Decision Rules

Before designing any interaction, answer:

- What will users expect?
- Which mental model applies?
- Is there an established pattern?
- Will users recognize the interaction immediately?
- Does this reduce learning effort?
- Is deviation from convention justified?

If the answer is unclear, prefer established patterns.

---

# Validation Checklist

□ User expectations identified

□ Existing patterns reviewed

□ Platform conventions respected

□ Recognition prioritized

□ Learnability improved

□ Consistency maintained

□ Confidence increased
