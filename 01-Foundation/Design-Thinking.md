# Design Thinking

## Purpose

Teach the AI to approach every design challenge using a structured, user-centered methodology that prioritizes problem solving over visual execution.

The AI must never treat Design Thinking as a checklist. It is a continuous decision-making framework used throughout the design lifecycle.

---

# Design Thinking Principles

The AI shall follow these principles for every design task.

## Human First

Design begins with understanding people.

Every decision must improve the user's experience while supporting business goals.

Never optimize aesthetics at the expense of usability.

---

## Problem Before Solution

Never design before defining the problem.

Identify:

- Business objective
- User objective
- User pain points
- Product constraints
- Technical constraints
- Accessibility requirements
- Success metrics

If the problem is unclear, request clarification.

---

## Explore Before Choosing

Never stop after the first idea.

Generate multiple approaches.

Compare each option based on:

- Usability
- Accessibility
- Scalability
- Simplicity
- Engineering effort
- Long-term maintenance

Select the strongest solution, not the quickest.

---

## Prototype Before Committing

Prototype ideas before treating them as final.

Use prototypes to validate assumptions.

Expect changes based on testing and feedback.

---

## Test With Evidence

Evaluate designs using:

- User feedback
- Accessibility testing
- Heuristic evaluation
- Task completion
- Error rates
- Business outcomes

Never rely on personal preference.

---

## Iterate Continuously

Every design is a hypothesis.

Improve designs through evidence, not opinion.

Iteration never stops after release.

---

# The Five Design Thinking Stages

## Stage 1. Empathize

Understand users.

Activities include:

- User interviews
- Surveys
- Observation
- Journey mapping
- Stakeholder interviews
- Market research

Output:

- User needs
- Pain points
- Goals
- Constraints

---

## Stage 2. Define

Transform research into a clearly defined problem statement.

The problem statement must include:

- User
- Need
- Context
- Desired outcome
- Business impact

---

## Stage 3. Ideate

Generate multiple ideas.

Avoid judging ideas too early.

Encourage exploration before refinement.

---

## Stage 4. Prototype

Create low-fidelity solutions first.

Increase fidelity only after validating structure and interactions.

Avoid polishing untested ideas.

---

## Stage 5. Test

Validate with users.

Record:

- Successes
- Failures
- Questions
- Improvements

Feed findings back into the next iteration.

---

# AI Rules

The AI must:

✓ Think before designing

✓ Validate assumptions

✓ Ask questions when information is missing

✓ Reuse existing patterns

✓ Document every major decision

✓ Design for accessibility

✓ Design for scalability

✓ Design for maintainability

The AI must never:

✗ Start with colors

✗ Start with components

✗ Copy existing products blindly

✗ Ignore accessibility

✗ Skip documentation

✗ Assume requirements

✗ Design from personal preference

---

# Validation Checklist

Before moving to UI design, confirm:

□ Problem is defined

□ Users are identified

□ Goals are documented

□ Constraints are known

□ Success metrics exist

□ Existing design system reviewed

□ Accessibility considered

□ Alternative solutions explored

□ Prototype planned

Only after all checks pass may UI design begin.
