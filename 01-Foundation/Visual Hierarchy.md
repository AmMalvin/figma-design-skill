# Visual Hierarchy

## Purpose

This module teaches the AI how to control attention through layout, typography, spacing, color, scale, and visual weight.

Visual hierarchy determines what users notice first, second, and last.

Every interface should communicate importance before users read the content.

---

# Definition

Visual Hierarchy is the deliberate arrangement of interface elements to guide attention, communicate importance, improve comprehension, and support decision making.

Good hierarchy reduces cognitive effort.

Poor hierarchy creates confusion.

---

# Objectives

After completing this module, the AI should be able to:

- Direct user attention
- Build clear layouts
- Improve readability
- Improve scanning
- Increase comprehension
- Reduce visual noise
- Improve conversion
- Improve usability
- Create balanced interfaces

---

# Visual Hierarchy Mindset

Never ask:

"Does this screen look attractive?"

Always ask:

- What should users notice first?
- Is the visual order intentional?
- Does every element deserve its prominence?
- Can users understand the page within seconds?
- Is unnecessary visual competition removed?

---

# Core Principles

Always:

- Establish one primary focal point.
- Create a clear reading order.
- Use spacing intentionally.
- Build consistent hierarchy.
- Reduce competing emphasis.
- Support scanning behavior.

Never:

- Give everything equal importance.
- Create multiple competing primary actions.
- Overuse color.
- Overuse large typography.
- Fill every space.

---

# Hierarchy Goals

Every interface should answer:

- What is this?
- Why does it matter?
- What should happen next?
- Where should the user focus?
- Which action has the highest priority?

---

# Human Scanning Behavior

People scan before they read.

Users should understand the screen within a few seconds.

The hierarchy should communicate meaning before detailed reading begins.

---

# AI Decision Rules

Before approving any screen:

- Is there one clear focal point?
- Is the reading order obvious?
- Is importance communicated visually?
- Is clutter removed?
- Does spacing improve comprehension?
- Can users scan the page quickly?

If any answer is negative, redesign the layout.

---

# Validation Checklist

□ Primary focal point defined

□ Reading order established

□ Visual clutter reduced

□ Hierarchy consistent

□ Scanning supported

□ Readability reviewed

□ Accessibility reviewed

□ Documentation updated
